CareTrack Database Schema

1. User (Custom User Model)

Extends standard Django AbstractUser. Handles authentication and role-based access.

Field NameData TypeKeyDescription
idAuto Increment INTPKUnique identifier for the user.
usernameVARCHAR(150)UniqueRequired. 150 characters or fewer. Letters, digits and @/./+/-/_ only.
passwordVARCHAR(128)Hashed password.
emailVARCHAR(254)User's email address.
first_nameVARCHAR(150)User's first name.
last_nameVARCHAR(150)User's last name.
roleVARCHAR(10)User role (Admin, Doctor, Nurse, Patient). Default: Patient.
phone_numberVARCHAR(15)User's contact number.
is_approvedBOOLEANIndicates if the user account is approved by admin. Default: False.
requires_password_changeBOOLEANFlags if user needs to change password on next login. Default: False.
is_activeBOOLEANDesignates whether this user should be treated as active.
is_staffBOOLEANDesignates whether the user can log into the admin site.
date_joinedDATETIMEDate the account was created.

2. DoctorProfile

Stores additional information specific to doctors.

Field NameData TypeKeyDescription
idAuto Increment INTPKUnique identifier.
user_idINTFKOne-to-One link to User table. Deletes profile if User is deleted.
specializationVARCHAR(100)Doctor's medical specialty.
license_numberVARCHAR(50)Doctor's medical license number.
hospital_clinic_nameVARCHAR(255)Name of the affiliated hospital or clinic.

3. NurseProfile

Stores additional information specific to nurses.

Field NameData TypeKeyDescription
idAuto Increment INTPKUnique identifier.
user_idINTFKOne-to-One link to User table. Deletes profile if User is deleted.
departmentVARCHAR(100)Hospital department the nurse is assigned to.
qualificationVARCHAR(100)Nurse's medical qualification.
employee_idVARCHAR(50)Internal employee identification number.

4. PatientProfile

Stores demographic and basic medical information for patients.

Field NameData TypeKeyDescription
idAuto Increment INTPKUnique identifier.
user_idINTFKOne-to-One link to User table. Deletes profile if User is deleted.
date_of_birthDATEPatient's date of birth.
ageINTPatient's age.
genderVARCHAR(10)Patient's gender.
addressTEXTPatient's residential address.
contact_numberVARCHAR(15)Primary contact number.
emergency_contactVARCHAR(15)Emergency contact number.
assigned_doctor_idINTFKLink to User table (Doctor). Can be null. Sets to NULL on deletion.
basic_health_issueTEXTDescription of primary health issues.
treatment_detailsTEXTCurrent or past treatment information.

5. FollowUpPlan

Defines the scheduled follow-up routines for a patient.

Field NameData TypeKeyDescription
idAuto Increment INTPKUnique identifier.
patient_idINTFKOne-to-One link to PatientProfile. Deletes plan if Patient is deleted.
start_dateDATEDate the follow-up plan started. Auto set on creation.
duration_daysINTDuration of the follow-up plan in days. Default: 30.
check_in_frequencyVARCHAR(50)How often the patient should check in (e.g., 'Daily'). Default: 'Daily'.
is_activeBOOLEANWhether the plan is currently active. Default: True.

6. DailyCheckIn

Records the daily health vitals submitted by patients.

Field NameData TypeKeyDescription
idAuto Increment INTPKUnique identifier.
patient_idINTFKLink to PatientProfile.
dateDATEDate of the check-in. Auto set on creation.
blood_pressure_systolicINTSystolic blood pressure reading.
blood_pressure_diastolicINTDiastolic blood pressure reading.
blood_sugarDECIMAL(5,2)Blood sugar level.
pulseINTHeart rate/pulse.
pain_levelINTPain level scale (1-10).
symptomsTEXTAny subjective symptoms reported by the patient.
created_atDATETIMETimestamp when the record was created.

7. Alert

System-generated or manual alerts based on critical patient check-ins.

Field NameData TypeKeyDescription
idAuto Increment INTPKUnique identifier.
daily_checkin_idINTFKLink to the DailyCheckIn that triggered the alert.
severityVARCHAR(10)Severity level: Low, Medium, High.
reasonVARCHAR(255)Reason for the alert (e.g., High Blood Pressure).
statusVARCHAR(15)Status: New, In Progress, Resolved. Default: New.
created_atDATETIMETimestamp when the alert was created.
handled_by_idINTFKLink to User (Doctor/Nurse) handling the alert. Set to NULL if deleted.

8. Message

Internal communication messages between users (Doctor <-> Patient, Nurse <-> Doctor).

Field NameData TypeKeyDescription
idAuto Increment INTPKUnique identifier.
sender_idINTFKLink to User sending the message.
receiver_idINTFKLink to User receiving the message.
contentTEXTBody of the message.
timestampDATETIMETimestamp when the message was sent.
is_readBOOLEANWhether the message has been read by the receiver. Default: False.
